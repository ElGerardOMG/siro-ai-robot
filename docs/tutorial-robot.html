Tutorial: Crear un robot

Guía rápida para definir un nuevo robot usando la estructura de robots/. Puedes copiar robots/darth_vader o robots/charro_negro como plantilla.

1. Estructura de carpetas

Crea una carpeta dentro de robots/, por ejemplo mi_robot/, con:

  • config.json — Configuración del robot (IA, voz, banderas, idle, etc.).
  • secrets.json — Credenciales (p. ej. google_api_key). No subas este archivo al repositorio.
  • ComponentNameDefinition.py — Enums de componentes (servos, LEDs, audios) que heredan de ComponentEnum.
  • SequenceNameDefinition.py — Diccionario nombre → { description, sequence } para animaciones.
  • sequences/ — Módulos Python que definen listas de movimientos (una lista por animación).
  • audios/ — Archivos WAV para idle y secuencias (opcional).
  • main.py — Punto de entrada: carga config, instancia IA, reconocimiento, sintetizador, controladores, secuenciador, director y Robot; asigna parámetros y llama a main.start().

2. Definir componentes (ComponentNameDefinition.py)

Cada enum representa un conjunto de componentes manejados por un mismo controlador. Sintaxis por miembro:

  • Servos/LEDs: NOMBRE = (channel, min_value, max_value)
  • Audios (para AudioPlayerController): NOMBRE = (channel, 0, 1, ruta_wav) — el cuarto valor es el label (ruta del archivo).

Ejemplo (Darth Vader):

class Servos(ComponentEnum):
    NECK_X = (1, 0, 180)
    SHOULDER_L_X = (3, 0, 180)
    ...

class Leds(ComponentEnum):
    EYES = (0, 0, 255)
    LIGHT_SABER = (1, 0, 255)

class AudioNames(ComponentEnum):
    SABER_ON = (0, 0, 1, f"{BASE_DIR}/audios/saber_on.wav")
    ...

3. Definir secuencias (sequences/)

Cada archivo en sequences/ (excepto __init__.py) puede exportar listas. El __init__.py descubre los módulos y rellena el diccionario SEQUENCES (nombre de la variable → lista).

Cada secuencia es una lista de pasos. Cada paso es un dict:

  • "parameters": { "time": segundos, "type": "linear" }
  • "components": { ComponentEnum: valor_numérico o None } — Si el valor es None, se aplica de inmediato.

Ejemplo (Ademan_1.py):

from ..ComponentNameDefinition import *

Ademan_1 = [
  { "parameters": { "time": 0.9, "type": "linear" },
    "components": {
      Servos.NECK_X : 0,
      Servos.SHOULDER_L_X : 0,
      ...
    },
  },
  { "parameters": { "time": 0.7, "type": "linear" },
    "components": { ... },
  },
]

4. SequenceNameDefinition.py

Asocia un nombre (el que la IA usará en las banderas) con descripción (para el prompt) y la secuencia (desde SEQUENCES):

from .sequences import SEQUENCES

SequenceNameDefinition = {
    "ADEMAN_1": {
        "description": "Ademán simple mientras habla 1",
        "sequence": SEQUENCES.get("Ademan_1")
    },
    "IDLE": { "description": "No hace nada", "sequence": SEQUENCES.get("Idle") },
    "FIN ACEPTA": { "description": "Animación de término cuando acepta", "sequence": SEQUENCES.get("SoundEffect") },
    ...
}

Las entradas con description se añaden al prompt de la IA (salvo las que estén en excluded_sequences_from_ai_prompt en config.json).

5. config.json (resumen)

  • ai — instructions, first_model_message, model.
  • finalization_flags — Lista de banderas que terminan la conversación (ej. ["FIN ACEPTA", "FIN RECHAZA"]).
  • robot_name, vosk_model_path.
  • voice — engine, rutas onnx/voices, voice, language, speed (Kokoro).
  • audio_filter — pitch_semitones, echo_delay, gain, etc.
  • speech_replacements — Diccionario de reemplazos para WordReplacerWrapper.
  • idle — sound_wavs, play_audio_interval (para idle_behaviour).
  • excluded_sequences_from_ai_prompt — Nombres que no se añaden al prompt (IDLE, FIN ACEPTA, etc.).
  • idle_sequence — Nombre de la secuencia idle (ej. "IDLE").

6. main.py — Orden típico

  1. Configurar logging y BASE_DIR.
  2. Cargar config.json y secrets.json.
  3. Construir IA (instrucciones + primer mensaje). Añadir al prompt las descripciones de las animaciones (excluyendo las de excluded_sequences_from_ai_prompt).
  4. Crear reconocimiento de voz (Vosk o DummyTextRecognition).
  5. Crear fábrica de audio y opcionalmente AudioFactoryFilter; crear sintetizador (Kokoro) y opcionalmente WordReplacerWrapper.
  6. Crear controladores para cada enum (Servos, Leds, AudioNames...) e inicializarlos con sus enums.
  7. Crear DefaultMoveSequencer, DialogueDirector(synthetizer, sequencer, SequenceNameDefinition), Robot(aiModel, recognition, sequencer, inputA, director, SequenceNameDefinition).
  8. Asignar robotName, initial_dialogue_text, end_dialogue_flags, idle_animation_name, idle_behaviour (función opcional en bucle, p. ej. sonidos cada X segundos).
  9. Llamar a main.start().

Puedes revisar robots/darth_vader/main.py y robots/charro_negro/main.py como referencia completa.

7. Qué controlador usar

Según tu hardware: MockController (pruebas), ArduinoSerialController (Arduino), AdafruitServoController (Raspberry + PCA9685), LedController (GPIO), RemoteUDPController (red). Para sonidos en secuencias, AudioPlayerController. Ver Controladores.
