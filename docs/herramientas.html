Herramientas

Las herramientas se encuentran en la carpeta tools/. Están enfocadas en la creación de animaciones y el testeo de controladores.

SampleServosDefinition.py

Implementación de ejemplo de ComponentEnum con una definición de servos. Sirve para usar en el resto de herramientas o para hacer pruebas rápidas sin definir tu propio enum.

ControllerTester

Herramienta con interfaz en consola para mover y cambiar los valores de los componentes. Útil para comprobar que un controlador y sus componentes funcionan correctamente. Los controles se activan con el teclado.

Cómo ejecutarla

python -m tools.ControllerTester

Configuración

Edita el archivo tools/ControllerTesterConfig.py. Contiene una única función get_config() que debe devolver los ComponentEnum que se usarán. Esos enums deben estar inicializados con el controlador que quieres probar (por ejemplo, creas MockController(Servos) o RemoteUDPController(Servos, ...) en esa función y luego devuelves el tipo de enum, p. ej. Servos). El controlador que uses al inicializar los componentes será el que se esté probando.

Ejemplo en ControllerTesterConfig.py:

from ai_talking_robot.controllers.MockController import MockController
from .SampleServosDefinition import Servos

def get_config():
    MockController(Servos)  # inicializa Servos con MockController
    return Servos

VirtualRobotSimulator

Simulador de un robot con extremidades y servomotores ya predefinidos (los mismos de SampleServosDefinition.py). El robot virtual se abre en el navegador y se puede mover con el ratón:

  • Shift + clic — Mover la vista.
  • Clic derecho — Rotar.
  • Rueda del ratón — Alejar o acercar.

Recibe instrucciones en localhost por el puerto 5005, por lo que se puede controlar con RemoteUDPController apuntando a 127.0.0.1:5005.

Cómo ejecutarla

python -m tools.VirtualRobotSimulator

Si usas AnimationCreator o ControllerTester con RemoteUDPController a ese puerto, el robot virtual reflejará los movimientos.

AnimationCreator

Herramienta similar a ControllerTester pero más robusta: sirve para crear animaciones completas. Incluye una línea de tiempo con los pasos de la animación: puedes agregar pasos nuevos o borrar existentes, reproducir la animación para previsualizarla, y guardar o cargar animaciones.

Cómo ejecutarla

python -m tools.AnimationCreator

Configuración

Edita tools/AnimationCreatorConfig.py. La función get_config() debe devolver dos cosas: primero un secuenciador (por ejemplo DefaultMoveSequencer), y segundo los ComponentEnum inicializados con el controlador que quieras usar. Ejemplo con el simulador virtual:

from ai_talking_robot.sequencer.DefaultMoveSequencer import DefaultMoveSequencer
from ai_talking_robot.controllers.RemoteUDPController import RemoteUDPController
from .SampleServosDefinition import Servos

def get_config():
    format = { "servo": "name", "angle": "value" }
    RemoteUDPController(Servos, format, "127.0.0.1", 5005)
    sequencer = DefaultMoveSequencer()
    return sequencer, Servos

Guardar y cargar animaciones

  • Al guardar: El archivo generado no incluirá el import de los ComponentEnum. Tendrás que añadirlo manualmente (p. ej. from ..ComponentNameDefinition import *) cuando copies la animación a tu robot.
  • Al cargar: El archivo debe ser válido (sin errores de sintaxis) y los ComponentEnum usados en la animación deben estar importados en ese archivo.

Combinar VirtualRobotSimulator con AnimationCreator (usando RemoteUDPController al puerto 5005) permite crear animaciones sin hardware real. Siempre que sea posible se recomienda validar después con el hardware real.

Calibración del VirtualRobotSimulator

Si tu robot real es parecido al del simulador y quieres que las posiciones coincidan, conviene calibrar. Dentro de VirtualRobotSimulator.py hay un diccionario current_angles:

  • El segundo valor de cada componente es el desfase: ajústalo para que la posición “0” del robot virtual coincida con la posición “0” del robot real (por ejemplo, cuando en ControllerTester o AnimationCreator un brazo marque 0, que el servo real en 0 se vea igual que en el simulador).
  • El tercer valor indica la dirección (1 o -1). Cámbialo a -1 si tu servo se mueve al revés. Ten en cuenta que muchos controladores no admiten valores negativos.

Dephaser

Cuando tienes animaciones hechas con un hardware (o con el VirtualRobotSimulator) y al usarlas en otro hardware se comportan distinto, suele haber un desfase: el “0” de un sistema no corresponde al mismo ángulo físico que el “0” del otro. Dephaser abre una carpeta con archivos de animación y les aplica un desfase e inversión por componente.

Cómo ejecutarla

python -m tools.Dephaser

Uso

  1. Componentes: Al iniciar, pide la ruta completa del archivo .py donde están declarados tus ComponentEnum (por ejemplo el ComponentNameDefinition.py de tu robot).
  2. Ajuste: Se muestra una interfaz en consola controlada por teclado para ajustar los valores de desfase y la inversión (YES/NO) de cada componente por separado. El desfase es la diferencia de ángulo entre el “0” del hardware anterior y el “0” del nuevo, tomando como positiva la dirección en que aumentaba el valor en el hardware anterior. Marca YES para invertir si el nuevo componente se mueve al lado contrario.
  3. Carpeta: Al final pide la carpeta donde están las animaciones (.py) a las que quieres aplicar el desfase. Dephaser modifica esos archivos.

Importante: Dephaser hace cambios irreversibles en los archivos. Se recomienda hacer una copia de las animaciones antes de usarlo.
