Módulos del proyecto

El código principal está en src/ai_talking_robot. Cada carpeta es un módulo con una responsabilidad clara.

integration

  • Robot.py — Clase principal. Recibe IA, reconocimiento, secuenciador, activador de entrada, director de diálogo y diccionario de animaciones. Ejecuta el bucle principal y el conversacional.
  • DialogueDirector.py — Convierte texto en DialogueObject, sintetiza voz y lanza animaciones en el secuenciador durante la reproducción del diálogo.
  • DialogueObject.py — Objetos de diálogo: lista de ítems (animación + texto + audio y estado).
  • CustomFormatter.py — Formato de logging para consola.

sequencer

  • ComponentEnum.py — Enum base que representa un componente (canal, min, max, label opcional). Actúa como puente al controlador: initialize(controller) vincula el enum a un AComponentController; la propiedad currentValue delega en get/set del controlador. Las animaciones trabajan con componentes, no con controladores directamente.
  • DefaultMoveSequencer.py — Ejecuta secuencias (lista de diccionarios con parameters.time y components). Interpola linealmente en el tiempo y actualiza component.currentValue. Permite cancelar la ejecución con cancelExecution().

controllers

Controladores de hardware (o simulación). Cada uno recibe un tipo que hereda de ComponentEnum e inicializa sus miembros con component.initialize(self). Implementan setComponentValue(component, value) y getComponentValue(component). Ver Controladores para detalle.

speech

  • AVoiceSynthetizer — Clase abstracta. synthetize(text) devuelve un AAudio.
  • KokoroSynthetizer — Síntesis con Kokoro (modelo ONNX + voces). Configurable: voz, idioma, velocidad.
  • FakeYouVoiceSynthetizer — Síntesis vía API FakeYou (marcado como deprecado).
  • WordReplacerWrapper — Envuelve otro sintetizador; aplica un diccionario de reemplazos de palabras antes de sintetizar (útil para pronunciación, p. ej. nombres).

voice

  • ASpeechRecognition — Clase abstracta. startRecognition() devuelve el texto reconocido.
  • VoskSpeechRecognition — Reconocimiento con modelo Vosk (micrófono, timeout de silencio).
  • DummyTextRecognition — Pide texto por input() en consola; útil para pruebas sin micrófono.

ai

  • AAIModelController — Clase abstracta. Gestiona conversación y envío de mensajes.
  • GoogleAIModel — Implementación con API de Google AI (p. ej. Gemma).

audioplayer

Abstracciones para crear y reproducir audio: AAudio, AAudioFactory, SimpleAudio, SimpleAudioFactory, AudioFactoryFilter (pitch, eco, ganancia), y especificaciones como WavFileSpec, WavBytesSpec, NumpyArraySpec.

input

  • AInputActivator — Clase abstracta. Define cómo se activa el inicio de la conversación.
  • EnterInputActivator — Espera pulsar Enter.
  • VoiceRecognitionActivator — Espera una palabra/frase de voz para activar.
