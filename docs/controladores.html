Controladores de componentes

Los controladores traducen los valores de las animaciones (secuenciador) al hardware real. La interfaz común es AComponentController: recibe un enum que hereda de ComponentEnum y en el constructor llama a component.initialize(self) para cada miembro. Métodos a implementar: setComponentValue(component, value) y getComponentValue(component).

Implementaciones disponibles

Clase Uso
MockController Simula el controlador: guarda valores en un diccionario por canal. Útil en pruebas o sin hardware.
ArduinoSerialController Envía comandos por puerto serial. Usa una plantilla con placeholders (name, channel, value, min_value, max_value, label) y envía una línea por cada setComponentValue.
AdafruitServoController Control de servos con PCA9685 (ServoKit) en Raspberry Pi. El canal del enum es el canal del servo; el valor se interpreta como ángulo entre min_value y max_value.
LedController GPIO en Raspberry Pi. El channel del enum es el número de pin; valor distinto de 0 → HIGH, 0 → LOW.
RemoteUDPController Envía un JSON por UDP con los campos que definas (name, channel, value, etc.). Útil para controlar otro proceso o dispositivo en la red.
ParallelControllerWrapper Envuelve otro controlador y ejecuta setComponentValue en un hilo con cola y timeout, para no bloquear si el controlador se cuelga.
AudioPlayerController Los componentes son índices de una lista de AAudio. setComponentValue(component, 0) = stop; otro valor = playAudioAsync(). Sirve para sonidos dentro de secuencias.

Cómo decidir qué controlador usar

  • Sin hardware / pruebas: MockController.
  • Servos por Arduino: ArduinoSerialController con tu formato de string (ej. "S,{channel},{value}\n").
  • Servos en Raspberry con PCA9685: AdafruitServoController.
  • LEDs en Raspberry (GPIO): LedController.
  • Control remoto (otro programa, otra máquina): RemoteUDPController con el formato JSON que espere el receptor.
  • Reproducir sonidos en secuencias: AudioPlayerController con un enum cuyos miembros tengan label con la ruta del WAV (o audios creados por fábrica).
  • Controlador lento o bloqueante: Envolver con ParallelControllerWrapper.

Cómo implementar un controlador

  1. Crear una clase que herede de AComponentController.
  2. En el constructor recibir el tipo de enum (p. ej. Servos) y opcionalmente parámetros de conexión (puerto, IP, etc.). El padre ya hace for component in components: component.initialize(self).
  3. Implementar setComponentValue(component, value): usar component.channel, component.min_value, component.max_value, component.label según necesites y enviar el valor al hardware.
  4. Implementar getComponentValue(component): devolver el valor actual si el hardware lo permite, o un valor por defecto.

Cada robot define sus enums en ComponentNameDefinition.py (p. ej. Servos, Leds, AudioNames) y asigna un controlador por cada tipo. Ver Tutorial: Crear un robot y los robots en robots/darth_vader y robots/charro_negro como referencia.
