Controladores de componentes

Los controladores traducen los valores de las animaciones (secuenciador) al hardware real. La interfaz común es AComponentController: recibe un tipo que hereda de ComponentEnum; en el constructor se llama a component.initialize(self) para cada miembro del enum, vinculando así componentes y controlador.

Implementaciones disponibles

Clase Uso
MockController Simula el controlador: guarda valores en un diccionario por canal. Útil en pruebas o sin hardware.
ArduinoSerialController Envía comandos por puerto serial. Usa una plantilla con placeholders (name, channel, value, min_value, max_value, label) y envía una línea por cada setComponentValue.
AdafruitServoController Control de servos con PCA9685 (ServoKit) en Raspberry Pi. El canal del enum es el canal del servo; el valor se interpreta como ángulo entre min_value y max_value.
LedController GPIO en Raspberry Pi. El channel del enum es el número de pin; valor distinto de 0 → HIGH, 0 → LOW.
RemoteUDPController Envía un JSON por UDP con los campos que definas (name, channel, value, etc.). Útil para controlar otro proceso o dispositivo en la red.
ParallelControllerWrapper Envuelve otro controlador y ejecuta setComponentValue en un hilo con cola y timeout, para no bloquear si el controlador se cuelga.
AudioPlayerController Los componentes son índices de una lista de AAudio. setComponentValue(component, 0) = stop; otro valor = playAudioAsync(). Sirve para sonidos dentro de secuencias.

Cómo usar e instanciar cada controlador

En todos los casos, primero defines un enum que hereda de ComponentEnum en ComponentNameDefinition.py (p. ej. Servos, Leds). Luego creas una instancia del controlador pasando ese tipo de enum. El constructor del controlador ya inicializa cada miembro del enum con component.initialize(self), así que no tienes que llamar a initialize manualmente.

MockController

Para pruebas o cuando no hay hardware. Parámetros: el tipo de enum y un nombre opcional (para logs).

from ai_talking_robot.controllers.MockController import MockController
from .ComponentNameDefinition import Servos, Leds

servoController = MockController(Servos, "Servos")
ledController = MockController(Leds, "Leds")

RemoteUDPController

Envía un JSON por UDP. Parámetros: tipo de enum, diccionario de campos (qué atributos del componente enviar y con qué nombre en el JSON), IP y puerto.

from ai_talking_robot.controllers.RemoteUDPController import RemoteUDPController
from .ComponentNameDefinition import Servos

# Campos disponibles: name, channel, value, min_value, max_value, label
udp_format = { "servo": "name", "angle": "value" }
servoController = RemoteUDPController(Servos, udp_format, "127.0.0.1", 5005)

En cada setComponentValue se enviará un JSON con las claves del diccionario (ej. servo, angle) y los valores tomados del componente (name, value, etc.).

ArduinoSerialController

Necesitas un objeto Serial (p. ej. serial.Serial(puerto, baud_rate, timeout=...)) y una cadena de formato con placeholders: name, channel, value, min_value, max_value, label.

from serial import Serial
from ai_talking_robot.controllers.ArduinoSerialController import ArduinoSerialController
from .ComponentNameDefinition import Servos

serial = Serial("COM3", 9600, timeout=0.1)
# Ejemplo: enviar "canal,valor" por línea
servoController = ArduinoSerialController(Servos, serial, "{channel},{value}")

AdafruitServoController

Para Raspberry Pi con PCA9685 (ServoKit). Parámetros: tipo de enum y opcionalmente número de canales (por defecto 16).

from ai_talking_robot.controllers.AdafruitServoController import AdafruitServoController
from .ComponentNameDefinition import Servos

servoController = AdafruitServoController(Servos, componentCount=16)

LedController

GPIO en Raspberry Pi. Parámetros: tipo de enum y opcionalmente modo GPIO (GPIO.BCM o GPIO.BOARD). El channel de cada miembro del enum es el número de pin.

from ai_talking_robot.controllers.LedController import LedController
from .ComponentNameDefinition import Leds

ledController = LedController(Leds, gpio_mode=GPIO.BCM)

AudioPlayerController

Reproduce audios durante las secuencias. Puedes pasar una lista de AAudio ya creada, o pasar None y usar después createAudios(factory, spec) para que cargue los WAV desde el label de cada componente.

from ai_talking_robot.controllers.AudioPlayerController import AudioPlayerController
from ai_talking_robot.audioplayer.AudioSpec import WavFileSpec
from .ComponentNameDefinition import AudioNames

# Opción 1: lista de audios ya creados
# controllerAudios = [audioFactory.create(WavFileSpec(path=p)) for p in rutas]
# audioController = AudioPlayerController(AudioNames, controllerAudios)

# Opción 2: None y crear audios desde los label de AudioNames (cada miembro tiene label = ruta WAV)
audioController = AudioPlayerController(AudioNames, None)
audioController.createAudios(audioFactory, WavFileSpec(""))

Los channel del enum deben coincidir con el índice en la lista de audios.

ParallelControllerWrapper

Envuelve otro controlador. Parámetros: la instancia del controlador a envolver y opcionalmente max_time_out en segundos.

from ai_talking_robot.controllers.ParallelControlerWrapper import ParallelControllerWrapper

servoController = RemoteUDPController(Servos, udp_format, "127.0.0.1", 5005)
servoController = ParallelControllerWrapper(servoController, max_time_out=2.0)

Cómo decidir qué controlador usar

  • Sin hardware / pruebas: MockController.
  • Servos por Arduino: ArduinoSerialController con tu formato de string.
  • Servos en Raspberry con PCA9685: AdafruitServoController.
  • LEDs en Raspberry (GPIO): LedController.
  • Control remoto (otro programa, otra máquina): RemoteUDPController.
  • Sonidos en secuencias: AudioPlayerController.
  • Controlador lento o bloqueante: Envolver con ParallelControllerWrapper.

Cómo implementar un controlador (crear una nueva clase)

Si necesitas un controlador que no existe (por ejemplo otro protocolo o hardware):

  1. Crear una clase que herede de AComponentController.
  2. En el constructor recibir al menos el tipo de enum (p. ej. Servos). Llamar a super().__init__(components) para que se ejecute component.initialize(self) en cada miembro. Guardar referencias y abrir conexiones si aplica.
  3. Implementar setComponentValue(self, component, value): usar component.channel, component.min_value, component.max_value, component.label según necesites y enviar el valor al hardware.
  4. Implementar getComponentValue(self, component): devolver el valor actual si el hardware lo expone, o un valor por defecto (p. ej. el último valor enviado guardado en un dict).

No hace falta que el robot llame a initialize en los componentes: el constructor del controlador ya lo hace al recibir el tipo de enum. Ver los controladores existentes en src/ai_talking_robot/controllers/ como referencia.
