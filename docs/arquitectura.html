Inicio Arquitectura Módulos Controladores Tutorial: Crear un robot

Arquitectura del sistema

El punto de entrada es integration/Robot.py. Ahí se orquesta el bucle principal y el bucle conversacional.

Estados del robot

El robot tiene un estado (Status) que indica en qué fase está:

  • WAITING — Esperando que el usuario active la conversación (por ejemplo pulsar Enter o hablar).
  • LISTENING — Escuchando al usuario (reconocimiento de voz activo).
  • THINKING — Enviando el mensaje a la IA y esperando respuesta.
  • SPEAKING — Reproduciendo la respuesta (voz + animaciones).

Bucle principal

  1. Inicialización — Se sintetiza el diálogo inicial, se resuelve la animación idle y, si está definido, se arranca el comportamiento idle en un hilo (por ejemplo sonidos en bucle).
  2. Bucle infinito exterior:
    • Si hay animación idle, se ejecuta en el secuenciador.
    • Esperar activación — Estado WAITING; el activador de entrada (p. ej. Enter o voz) espera hasta que el usuario quiera empezar. Luego se reproduce el mensaje inicial y se inicia una nueva conversación en la IA.
    • Bucle conversacional (ver abajo). Al salir de este bucle (por bandera de finalización), se vuelve a idle y a esperar una nueva activación.

Bucle conversacional

1 Escuchar: Estado LISTENING. El reconocimiento de voz se activa hasta obtener texto (o (*silencio) si no se reconoce nada).
2 Pensar: Estado THINKING. Se envía el texto a la IA (con reintentos hasta maxAIAttempts).
3 Responder: La respuesta se parsea con DialogueDirector.parseTextToDialogue() y se obtiene un DialogueObject (lista de ítems: animación + texto).
4 Reproducir: Estado SPEAKING. DialogueDirector.playDialogue() sintetiza voz, reproduce audio y ejecuta la secuencia asociada a cada bandera en el secuenciador.
5 Salir: Si end_dialogue_flags está definido y alguna de esas banderas aparece en la respuesta, se hace break y se termina el bucle conversacional (vuelta a idle y espera de nueva inicialización).

Banderas en las respuestas de la IA

En el prompt se indica a la IA que formatee así:

  • [NOMBRE_ANIMACION] — Indica qué animación/gesto ejecutar antes (o durante) la frase que sigue. El nombre debe coincidir con una entrada en el diccionario de animaciones del robot (SequenceNameDefinition).
  • [FIN ACEPTA], [FIN RECHAZA], etc. — Banderas de finalización. Si alguna está en end_dialogue_flags, al aparecer en la respuesta se termina la conversación.

Ejemplo de respuesta esperada:

[ADEMAN_1] Soy Darth Vader.
[ADEMAN_2] Dime cuál es tu virtud.

De la respuesta al hardware

  1. Parseo: parseTextToDialogue(text) divide por el patrón [...] y arma ítems (animación + texto).
  2. Por cada ítem: Se sintetiza el texto a audio, se reproduce y se busca la secuencia por nombre. Se cancela la ejecución actual del secuenciador y se ejecuta la nueva secuencia (en hilo si hay audio, bloqueante si no).
  3. Secuenciador: Una secuencia es una lista de pasos; cada paso tiene parameters.time y components (ComponentEnum → valor). El secuenciador interpola en el tiempo y asigna component.currentValue = ....
  4. Componentes y controladores: Cada ComponentEnum está vinculado a un AComponentController. Al escribir currentValue, se llama a setComponentValue() del controlador, que envía el valor al hardware (serial, UDP, GPIO, etc.).
